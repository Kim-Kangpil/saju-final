# 배포·모바일 환경 체크리스트

## 1. 도메인·CORS (로그인/저장 실패 시 먼저 확인)

- **백엔드** Render 환경변수에 반드시 설정:
  - `FRONTEND_URL`: 사용자가 실제로 접속하는 주소와 동일 (예: `https://hsaju.com` 또는 `https://www.hsaju.com`)
  - `CORS_ORIGINS`: 접속 가능한 모든 주소를 콤마로 (예: `https://hsaju.com,https://www.hsaju.com`)
- **프론트**에는 `FRONTEND_URL` 불필요. 로그인·저장 동작은 **백엔드** CORS 설정에 따름.
- 서버 기동 시 로그에 `CORS 허용 origin: [...]` 출력됨. 배포 후 여기 포함 여부 확인.

## 2. 모바일에서 로그인 정보가 안 뜨는 경우

- **원인**: 프론트와 백엔드가 서로 다른 도메인이면, 모바일 브라우저가 세션 쿠키를 안 보낼 수 있음.
- **대응**:
  1. 위 1번대로 `FRONTEND_URL`·`CORS_ORIGINS`가 실제 접속 URL과 정확히 일치하는지 확인.
  2. 동일해도 이메일이 안 보이면 "이 기기에서는 이메일이 표시되지 않을 수 있음" 안내가 뜨며, 저장·목록은 동작할 수 있음.
  3. 근본 해결: API를 프론트와 같은 도메인으로 제공 (예: `api.hsaju.com`).

## 3. 세션 유지

- 로그인 후 세션 쿠키 만료: **30일**. 30일 이내 재방문 시 로그인 유지.

## 4. 사주 저장

- 저장된 사주는 **계정(user_id)별로 유지**되며, 로그인/재접속 시 초기화되지 않음.
