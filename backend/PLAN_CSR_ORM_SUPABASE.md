# Controller-Service-Repository + ORM + Supabase 마이그레이션 플랜

## 목표

1. **Controller - Service - Repository** 구조로 백엔드 재구성  
2. **ORM** 사용: 직접 `sqlite3`/connection 대신 **SQLModel** (FastAPI와 호환, Pydantic + SQLAlchemy)  
3. **DB 이전**: 로컬 SQLite 파일 → **Supabase(PostgreSQL)** 로 안전하게 이전  

---

## Part A. 기술 스택

| 구분 | 선택 | 비고 |
|------|------|------|
| ORM | **SQLModel** | FastAPI 제작자 동일, Pydantic + SQLAlchemy. 타입 힌트·스키마 재사용 좋음 |
| DB | **Supabase (PostgreSQL)** | 무료 티어, 연결 문자열만 있으면 사용 가능 |
| 연결 방식 | **async** (asyncpg) | FastAPI 비동기와 잘 맞음. SQLModel은 `create_async_engine` 지원 |

## Part C. Supabase 연동 및 스키마

### C.1 Supabase에서 할 일

1. [Supabase](https://supabase.com) 프로젝트 생성  
2. **Project Settings → Database** 에서 **Connection string** 복사  
   - 예: `postgresql://postgres.[ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres`  
   - **URI** 또는 **Session mode** 사용 (Transaction mode는 풀러 5432)  
3. **환경 변수**에 추가 (로컬 `.env`, Render 등)  
   - `DATABASE_URL=postgresql+asyncpg://postgres.[ref]:[password]@...`  
   - Supabase는 PostgreSQL이므로 `postgresql+asyncpg://` 형태로 사용 (SQLAlchemy async 드라이버)

### C.2 PostgreSQL 스키마 (현재 SQLite 기준 매핑)

| SQLite 테이블 | PostgreSQL 테이블 | 비고 |
|---------------|-------------------|------|
| users | users | id SERIAL PRIMARY KEY, provider, provider_id, email, nickname, created_at, last_login, seed_balance |
| saju | saju | id SERIAL, user_id INTEGER REFERENCES users(id), name, relation, birthdate, birth_time, calendar_type, gender, created_at |
| payments | payments | id SERIAL, user_id TEXT, payment_id, order_id, status, created_at |
| inquiries | inquiries | id SERIAL, name, email, subject, message, created_at |

- SQLite `INTEGER PRIMARY KEY AUTOINCREMENT` → PostgreSQL `SERIAL PRIMARY KEY` 또는 `GENERATED BY DEFAULT AS IDENTITY`  
- `TEXT` → `VARCHAR` 또는 `TEXT` 그대로  
- 인덱스: `users(provider, provider_id)` UNIQUE, `saju(user_id)`, `payments(order_id)` UNIQUE

## Part E. 스키마 상세 (PostgreSQL DDL 예시)

Supabase SQL Editor에서 실행할 수 있는 예시입니다.

```sql
-- users
CREATE TABLE IF NOT EXISTS users (
    id SERIAL PRIMARY KEY,
    provider VARCHAR(64) NOT NULL,
    provider_id VARCHAR(255) NOT NULL,
    email VARCHAR(255),
    nickname VARCHAR(255),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_login TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    seed_balance INTEGER NOT NULL DEFAULT 0,
    UNIQUE(provider, provider_id)
);
CREATE UNIQUE INDEX IF NOT EXISTS idx_users_provider_provider_id ON users(provider, provider_id);

-- saju
CREATE TABLE IF NOT EXISTS saju (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id),
    name VARCHAR(255) NOT NULL,
    relation VARCHAR(255),
    birthdate VARCHAR(32) NOT NULL,
    birth_time VARCHAR(32),
    calendar_type VARCHAR(32) NOT NULL,
    gender VARCHAR(32) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_saju_user_id ON saju(user_id);

-- payments
CREATE TABLE IF NOT EXISTS payments (
    id SERIAL PRIMARY KEY,
    user_id VARCHAR(64) NOT NULL,
    payment_id VARCHAR(255) NOT NULL,
    order_id VARCHAR(255) NOT NULL UNIQUE,
    status VARCHAR(64) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- inquiries (contact)
CREATE TABLE IF NOT EXISTS inquiries (
    id SERIAL PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL,
    subject VARCHAR(255) NOT NULL,
    message TEXT NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```
