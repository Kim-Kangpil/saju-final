## 1단계: 세션 쿠키 만료 연장

- **목적**: 로그인 후 장시간 로그아웃하지 않고 다시 들어와도 유지되도록.
- **변경**:
  - `backend/auth_kakao.py`: `hsaju_session` 쿠키 `max_age` **7일 → 30일**
  - `backend/auth_google2.py`: 동일 **7일 → 30일**
- **효과**: 한 번 로그인 후 30일 이내 재방문 시 별도 로그인 없이 유지.

---

## 2단계: 모바일/도메인 설정 가이드 정리

- **목적**: 배포·모바일 이슈 시 한 곳에서 확인할 수 있도록.
- **추가 파일**: `backend/DEPLOY_CHECKLIST.md`
  - 도메인·CORS 설정 (FRONTEND_URL, CORS_ORIGINS) 확인 방법
  - 모바일에서 로그인 정보가 안 뜰 때 원인·대응
  - 세션 유지 기간(30일) 안내
  - 사주 저장이 계정별로 유지된다는 안내

---

## 3단계: 로그인·저장 메시지 일관성

- **목적**: 사용자에게 보이는 문구 통일.
- **변경**:
  - `frontend/app/saju-add/page.tsx`: "저장에 실패했습니다" → "저장에 실패했습니다." (마침표 통일, 2곳)
- **기존 유지**:
  - 401 시: "로그인 상태를 서버에서 확인하지 못했습니다. 서버의 FRONTEND_URL·CORS_ORIGINS가 실제 접속 주소와 일치하는지 확인해 주세요."
  - 마이페이지 fallback: "로그인됨 (이 기기에서는 이메일이 표시되지 않을 수 있습니다. 저장·목록 기능은 정상 이용 가능합니다.)"
